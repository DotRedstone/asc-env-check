## 遇到的问题

请在这里补充你遇到的问题、完整报错和解决方法。

- 问题 1：WSL2 内无法直接访问 github.com，`git clone` 卡在 "Cloning into 'asc-env-check'..." 后超时失败。
  - 报错：`Cloning into 'asc-env-check'...`（随后无响应，被 timeout 杀掉）。WSL 内 `curl -sSI https://github.com` 也无响应，但 Windows 主机端可以访问。
  - 解决方法：在 Windows 主机侧完成 `git clone --depth=1 https://github.com/DotRedstone/asc-env-check.git`，再把仓库目录放进 WSL2 的 `~/asc-env-check`。后续所有编译/运行都在 WSL2 内完成。


- 问题 2：conda 安装的 OpenMPI 的 `mpicc` 包装器默认调用 conda 自带编译器 `x86_64-conda-linux-gnu-cc`，但该编译器未安装，导致 `make mpi` 失败。
  - 报错：`The Open MPI wrapper compiler was unable to find the specified compiler x86_64-conda-linux-gnu-cc in your PATH.`
  - 解决方法：设置 `OMPI_CC=gcc`、`OMPI_CXX=g++` 让 OpenMPI 包装器改用系统 gcc/g++。为让脚本 `bash -lc` 调用也能生效，把 `export OMPI_CC=gcc` 及 conda 初始化写入 `~/.bashrc` 的交互式守卫（`case $- in ... return`）**之前**，确保非交互式 shell 也能加载。

- 最终结果：普通 C / OpenMP / MPI 三项测试全部通过（exit code 均为 0）。
