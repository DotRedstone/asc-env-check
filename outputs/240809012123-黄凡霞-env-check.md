## 遇到的问题
- 问题1：初次克隆仓库时本地已经存在asc-env-check同名目录，git克隆操作报错无法覆盖
- 解决方法：克隆自己Fork得到的仓库时指定新文件夹名asc-env-check-new，规避目录重名冲突

- 问题2：初始状态终端识别不到python指令，Python检测返回exit code 1
- 解决方法：激活conda的base环境，启用Miniconda内置的Python解释器，重跑脚本检测恢复正常

- 问题3：WSL系统默认的软件镜像源为国外官方源，镜像源版本适配不佳，下载apt依赖时网速缓慢，还出现包版本不匹配、依赖校验失败的报错
- 解决方法：依据本机Ubuntu的具体版本，更换适配的国内镜像源（例如清华镜像源），更新apt缓存`sudo apt update`之后再进行依赖安装，解决版本适配异常、下载卡顿的问题
